Project Overview

The KUWeb Design System was developed for Koç University to support a new website theme with a focus on consistency, accessibility, and scalability. Built in Figma, it includes reusable components, design tokens, and clear documentation to streamline collaboration between designers and developers. The system ensures a unified brand experience, reduces design time, and supports long-term maintainability across the university’s digital platforms.

Project Goal

Develop a scalable design system for the university's new website theme to improve visual consistency and streamline design workflows.

My Role

I was the lead UX/UI designer responsible for building the design system from the ground up. I defined tokens, components, and documentation in Figma, ensuring accessibility and responsiveness across devices. I worked closely with developers to support implementation.

This design system ensures a unified brand voice and visual identity across the Koç University Websites New Theme project, enabling a seamless experience. By integrating design and content standards, this system highlights the creation of cohesive, user-friendly, and accessible web experiences.

Key Metrics

The Challenge

A new project meant a brand new design system. The old theme had an outdated look that did not match the style the university was aiming for. This meant designing concise elements that can aid designers, content creators and developers.

I needed to: Create master components, Gather use cases for the existing old theme, Establish style guides, Ensure design consistency across pages, Create brand and content guidelines, Ensure accessibility

The Benefits

One source for all components, patterns, and styles.

Consistent design across pages.

Ability to replicate designs quickly with pre-made components and elements.

Reducing work time for designers, content creators and developers.

Clear guide for website administrators.

The Impact

The KUWeb Design System refined the design process for the university’s digital platform, reducing design time by 28% and ensuring consistency across multiple applications. By providing a scalable, reusable set of components, the system improved collaboration between design, content and development teams. The system also facilitated easier updates and maintenance, significantly enhancing the platform's long-term sustainability.

The Approach

I started creating the design systems documentation from scratch at the beginning of the project. My UX team members helped me with the framework and documentation.

I researched industry standards for web design and design systems to create a concise system. With clear rules, I established new styles. I then implemented UX writing suggestions on content creation. Every component and module design is made after assessing user needs.

Design System consists of

  • In-page annotations explaining the use of tools
  • Component Library: modular elements like buttons, icons etc.
  • Pattern Library: complex design elements and modules
  • Brand Style Guides: colouring, typography
  • Design Principles: basic design principles to follow and layout rules
  • Content Guidelines: tone of voice, audience, writing and visual image guides
  • Accessibility Guidelines: explanation of rules and usage of the accessibility tool

What I learned

As the lead designer on the KUWeb Design System, I honed my skills in building scalable design solutions. I learned how to document design patterns in a way that is easily accessible and reusable. By working closely with developers, I gained a deeper understanding of how design systems need to evolve and be maintained over time. This project reinforced the importance of communication, collaboration, and ensuring the system is adaptable to future needs.
